## 1. Algorithm Description

### 1.1 Overview

Aortica is an open-source, multi-task deep learning system for 12-lead ECG analysis. It performs simultaneous classification and risk prediction across four clinical task domains:

- **Rhythm & Conduction** (28 classes): detection of arrhythmias, conduction abnormalities, and rare rhythm disorders
- **Structural & Functional** (19 classes): screening for structural heart disease and functional abnormalities
- **Ischaemia & Metabolic** (19 classes): detection of ischaemic patterns, STEMI mimics, and metabolic/drug effects
- **Risk Prediction** (6 continuous outputs): prognostic scoring for mortality, heart failure, AF onset, ejection fraction, conduction disease, and sudden cardiac death

### 1.2 Architecture

The model employs a shared-backbone architecture:

1. **Backbone Encoder** (`AorticaBackbone`): 1D ResNet with residual blocks at 64, 128, 256 filter widths; adaptive pooling for variable input lengths
2. **Cross-Lead Attention** (`CrossLeadAttention`): 4-head, 64-dim multi-head attention capturing inter-lead relationships
3. **Task Heads**: Four independent classification/regression heads connected to the shared representation

### 1.3 Input/Output Specification

| Parameter | Specification |
|-----------|--------------|
| **Input** | 12-lead ECG signal, shape `[batch, 12, samples]` |
| **Sample Rates** | 250–1000 Hz (adaptive pooling) |
| **Window Length** | 2.5–10 seconds |
| **Output** | `MultiTaskOutput` with per-task sigmoid/regression values |
| **Calibration** | Temperature scaling applied post-hoc per task head |
| **Uncertainty** | Conformal prediction sets at configurable coverage level |

### 1.4 Signal Preprocessing Pipeline

1. Format-agnostic ingestion via universal reader (`read_ecg()`)
2. Signal denoising: baseline wander removal (0.5 Hz highpass), powerline notch (50/60 Hz), high-frequency lowpass (40 Hz)
3. Signal quality scoring: per-lead quality (0–100), overall accept/review/reject
4. Resampling to target rate (default 500 Hz)
5. Lead normalisation to standard 12-lead ordering

## 5. Known Limitations

### 5.1 Performance Limitations

- Model performance is bounded by training data distribution (PTB-XL)
- Rare arrhythmia subtypes have limited training examples and may have lower sensitivity
- Edge model (INT8 quantisation) introduces up to 3% performance degradation
- PDF/image-scanned ECGs have inherent digitisation error (Pearson r ≥ 0.85 target)

### 5.2 Demographic Limitations

- Training cohort is predominantly European (PTB-XL, single German centre)
- Performance may vary across ethnic groups not well represented in training data
- Equity gating enforces no statistically significant demographic performance gap (Bonferroni-corrected p<0.05) for classes with N>100 test examples

### 5.3 Clinical Limitations

- **Decision support only** — all outputs require clinician review and judgment
- Not validated for real-time continuous telemetry monitoring
- Not validated for paediatric populations
- Risk predictions are calibrated to training cohort priors

## 6. Test Methodology

### 6.1 Evaluation Protocol

Aortica uses a standardised benchmark harness (`aortica.evaluation.benchmark()`) that ensures:

1. **Deterministic evaluation**: fixed random seed produces identical results
2. **Held-out test set**: PTB-XL fold 10 (never used for training or validation)
3. **Full pipeline**: raw ECG → preprocessing → model inference → metric computation
4. **Comprehensive metrics**: per-class AUC, sensitivity, specificity, F1, macro-F1, ECE, C-index, Brier score

### 6.2 Metric Definitions

| Metric | Definition | Task Type |
|--------|-----------|-----------|
| **AUC** | Area under ROC curve (one-vs-rest) | Classification |
| **Sensitivity** | True positive rate (recall) at threshold 0.5 | Classification |
| **Specificity** | True negative rate at threshold 0.5 | Classification |
| **F1** | Harmonic mean of precision and recall | Classification |
| **Macro-F1** | Unweighted mean of per-class F1 scores | Classification |
| **ECE** | Expected Calibration Error (10 bins) | Classification |
| **C-index** | Concordance index for ranking accuracy | Risk |
| **Brier Score** | Mean squared error of predicted probabilities | Risk |

### 6.3 Statistical Testing

- Equity gate uses permutation test or bootstrap CI
- Bonferroni correction for multiple comparisons
- Minimum sample size N>100 per subgroup for significance testing
- Alpha = 0.05 (configurable)

### 6.4 Test Data Conditions

- Clean ECGs: standard clinical quality (quality score ≥70)
- Noisy ECGs: intentionally degraded signals (baseline wander, powerline noise)
- Edge cases: rare conditions, borderline confidence levels
