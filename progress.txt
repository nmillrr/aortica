# Progress Log

## Learnings
(Patterns discovered during implementation)
- Python 3.13 does not have PyTorch or neurokit2 available via pip. Made these optional dependencies (`[torch]`, `[signal]` extras).
- ECGRecord uses scipy.signal.resample (Fourier method) for resampling; scipy is already a core dependency.
- to_millivolts() supports µV, uV (ASCII), mV, and V unit strings.
- Use `setuptools.packages.find` with `include = ["aortica*"]` to auto-discover subpackages.

## Iteration 4 - US-004: WFDB Format Reader
- Implemented: `aortica.io.read_wfdb(path)` function that loads WFDB .hea/.dat files into ECGRecord
- Files changed:
  - aortica/io/wfdb_reader.py (new) - read_wfdb() with unit normalization (mV→µV), multi-segment support via m2s=True
  - aortica/io/__init__.py (modified) - exports read_wfdb and ECGRecord
  - tests/io/test_wfdb_reader.py (new) - 15 fast tests (synthetic WFDB fixtures) + 3 slow PhysioNet integration tests (MIT-BIH 100, PTB-XL 00001_hr)
- Learnings for future iterations:
  - wfdb.rdrecord() with m2s=True automatically flattens multi-segment records into a single Record
  - wfdb returns signals as [samples, channels] — must transpose to [leads, samples] for ECGRecord
  - wfdb records store physical units per-channel; reader normalizes all to µV using a lookup table
  - wfdb.wrsamp() can create synthetic WFDB records for testing without needing real data files
  - wfdb.dl_database() can download specific records from PhysioNet for integration tests

## Iteration 5 - US-005: CSV and MAT Format Readers
- Implemented: `read_csv()` with `CSVConfig` and `read_mat()` with `MATConfig`, both returning ECGRecord
- Files changed:
  - aortica/io/csv_reader.py (new) - read_csv() with CSVConfig dataclass, supports column-per-lead and row-per-lead orientations, time column exclusion, custom delimiter, skip rows
  - aortica/io/mat_reader.py (new) - read_mat() with MATConfig dataclass, supports leads_first/samples_first orientation, optional lead names and sample rate variables from MAT file
  - aortica/io/__init__.py (modified) - exports read_csv, CSVConfig, read_mat, MATConfig
  - tests/io/test_csv_mat_readers.py (new) - 27 tests covering both readers: basic loading, orientations, metadata, error handling, ECGRecord integration
- Learnings for future iterations:
  - scipy.io.loadmat is already available as scipy is a core dep — no new dependency needed for MAT files
  - numpy.genfromtxt is simpler than pandas for pure-numeric CSV reading
  - MATLAB cell arrays of strings come back as nested object ndarrays from loadmat — _extract_lead_names helper handles the various nested forms
  - CSVConfig and MATConfig use dataclasses with sensible defaults for clean API
  - Both readers follow the same pattern as wfdb_reader.py: validate path, load data, normalize to ECGRecord
